# AdForge Intent

## Purpose

This app exists to help me win the assessment interview for the Forward Deployed Engineer - Creative Technologist role.

AdForge should not read like a generic image-generation demo. It should present as a credible proof of concept for an embedded creative technologist who can:

- understand ambiguous client problems quickly
- design a defensible end-to-end solution
- prototype fast without losing technical rigor
- integrate GenAI into real creative workflows
- explain tradeoffs clearly to technical and non-technical stakeholders

## Assessment Context

The take-home exercise is about creative automation for scalable social campaigns. The implementation needs to show more than code that runs. It needs to demonstrate judgment:

- alignment to business goals
- understanding of creative operations pain points
- practical AI orchestration
- maintainable system design
- outputs that are easy to review, explain, and defend live

The real goal is not just "generate assets." The goal is to show how a Forward Deployed Engineer would approach a client problem from brief to outputs to validation to reporting.

## Role Lens

The Forward Deployed Engineer - Creative Technologist is a hands-on technical innovator who works with AI technology partners to rapidly prototype and deploy AI-powered creative solutions that maximize value from Generative AI investments.

The role operates at the intersection of design and engineering and owns systems end-to-end, from rapid prototype through deployment.

The role emphasizes:

- rapid prototyping and iteration
- Adobe ecosystem development
- custom tooling and workflow automation
- API orchestration and systems integration
- hands-on troubleshooting
- multimodal AI integration
- client-embedded technical leadership
- documentation, knowledge transfer, and reusable patterns

## What AdForge Must Demonstrate

### 1. Rapid prototyping with intent

AdForge should feel like a focused proof of concept built under real constraints, not a toy project or a grab bag of features.

It should show:

- a narrow but meaningful workflow
- fast local execution
- clear iteration paths
- deliberate scoping choices

### 2. Creative workflow understanding

AdForge should reflect how real campaign production works:

- campaign briefs drive output generation
- existing assets are reused when available
- new assets are generated only when needed
- placements and aspect ratios are treated as operational requirements
- copy, branding, and approvals matter as much as image generation

### 3. Explainable AI orchestration

The system should be easy to defend in a live interview:

- what inputs exist
- what decisions the pipeline makes
- when GenAI is used
- how outputs are validated
- where the system is deterministic versus heuristic
- what is mocked, what is real, and why

### 4. Integration mindset

Even if this assessment runs locally, AdForge should suggest production extensibility:

- storage abstraction
- model/provider abstraction
- structured campaign configuration
- output metadata and reporting
- clear seams for Adobe, cloud, or client-system integration

### 5. End-to-end ownership

The app should show that I can own a technical system from ideation through delivery:

- define the problem clearly
- encode business rules in the input contract
- implement the pipeline
- validate outputs
- document assumptions and limits
- present the result clearly

## Signals The Interview Should See

AdForge should communicate these traits:

- I can prototype quickly without producing fragile architecture.
- I can balance design taste with engineering discipline.
- I can turn vague business needs into a concrete, testable system.
- I know where GenAI adds value and where deterministic software should stay in control.
- I can create a solution that is maintainable, explainable, and extensible.
- I understand creative tooling, automation pipelines, and client-facing delivery.

## Technical Expectations To Reflect

The target role values experience across:

- Adobe Creative Cloud APIs, SDKs, App Builder, UXP, CEP, Creative SDKs, and Firefly Services
- Node.js, Python, modern web development, API integration, and cloud deployment
- LLMs, diffusion models, RAG pipelines, vector databases, prompt engineering, and multimodal systems
- plugin and workflow automation across tools like Photoshop and Figma
- analyzing unfamiliar systems and creating practical client integrations

AdForge does not need to implement all of that in this assessment. It does need to feel consistent with that background and show the same operating style.

## Product Intent

AdForge is a creative automation pipeline for localized social campaigns.

It should be framed as a system that:

- ingests a structured campaign brief
- resolves reusable assets first
- generates missing creative assets with AI
- produces placement-ready campaign variants
- applies brand and content controls
- emits reviewable outputs and reporting

That framing is stronger than presenting it as "a script that makes ad images."

## Design Principles

When making decisions in this repo, optimize for:

- clarity over cleverness
- explicit contracts over hidden assumptions
- modular boundaries over entangled orchestration
- reviewability over black-box automation
- defendable heuristics over inflated claims
- strong defaults over excessive configurability

## Non-Goals

For this interview project, AdForge does not need to be:

- fully productionized
- deeply scaled
- visually perfect in every output
- integrated with every Adobe surface
- a complete approval or analytics platform

It does need to be coherent, credible, and easy to explain.

## Standard For "Amazing"

An amazing assessment app is not the one with the most features.

It is the one that most clearly shows:

- good product judgment
- strong engineering fundamentals
- practical AI integration
- thoughtful workflow design
- clean storytelling in the code, docs, and demo

That is the standard AdForge should meet.
